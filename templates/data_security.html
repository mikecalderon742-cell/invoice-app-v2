{% if lang == 'es' %}1. Enfoque general{% else %}1. General Approach{% endif %}

{% if lang == 'es' %} BillBeam busca aplicar medidas administrativas, técnicas y operativas razonables para ayudar a proteger información contra acceso no autorizado, alteración, divulgación o destrucción. {% else %} BillBeam seeks to apply reasonable administrative, technical, and operational measures to help protect information against unauthorized access, alteration, disclosure, or destruction. {% endif %}

{% if lang == 'es' %} Estas medidas pueden revisarse y mejorarse con el tiempo según cambian el producto, la infraestructura y los riesgos conocidos. {% else %} These measures may be reviewed and improved over time as the product, infrastructure, and known risks evolve. {% endif %}

{% if lang == 'es' %}2. Acceso a cuentas{% else %}2. Account Access{% endif %}

{% if lang == 'es' %} El acceso a cuentas depende de credenciales de autenticación. Los usuarios son responsables de mantener sus credenciales confidenciales y evitar compartirlas con terceros no autorizados. {% else %} Account access depends on authentication credentials. Users are responsible for keeping their credentials confidential and avoiding sharing them with unauthorized third parties. {% endif %}

{% if lang == 'es' %}3. Seguridad en tránsito y proveedores{% else %}3. Security in Transit and Through Providers{% endif %}

{% if lang == 'es' %} BillBeam puede depender de proveedores externos de infraestructura, hosting, procesamiento de pagos, correo electrónico y servicios técnicos relacionados. Cuando corresponda, la información puede transmitirse mediante conexiones protegidas por tecnologías estándar del sector. {% else %} BillBeam may rely on third-party providers for infrastructure, hosting, payment processing, email delivery, and related technical services. Where appropriate, information may be transmitted using industry-standard protected connections. {% endif %}

{% if lang == 'es' %}4. Pagos{% else %}4. Payments{% endif %}

{% if lang == 'es' %} Los pagos pueden procesarse mediante proveedores externos como Stripe. BillBeam no está diseñado para almacenar números completos de tarjetas de pago en sus propios sistemas. {% else %} Payments may be processed through third-party providers such as Stripe. BillBeam is not designed to store full payment card numbers in its own systems. {% endif %}

{% if lang == 'es' %}5. Acceso interno mínimo necesario{% else %}5. Internal Need-to-Know Access{% endif %}

{% if lang == 'es' %} El acceso interno a información debe limitarse, en la medida razonable, a personas, sistemas o proveedores que necesiten esa información para operar, mantener o dar soporte al Servicio. {% else %} Internal access to information should be limited, to the extent reasonable, to persons, systems, or providers who need that information to operate, maintain, or support the Service. {% endif %}

{% if lang == 'es' %}6. Retención y eliminación{% else %}6. Retention and Deletion{% endif %}

{% if lang == 'es' %} BillBeam puede conservar información mientras sea necesario para operar el Servicio, cumplir fines operativos razonables, resolver disputas, aplicar acuerdos o cumplir obligaciones legales. {% else %} BillBeam may retain information as necessary to operate the Service, fulfill reasonable operational purposes, resolve disputes, enforce agreements, or comply with legal obligations. {% endif %}

{% if lang == 'es' %} Cuando sea apropiado y posible, los datos podrán eliminarse o anonimizarse conforme a políticas internas o solicitudes válidas del usuario. {% else %} Where appropriate and feasible, data may be deleted or anonymized in accordance with internal policies or valid user requests. {% endif %}

{% if lang == 'es' %}7. Respuesta ante incidentes{% else %}7. Incident Response{% endif %}

{% if lang == 'es' %} Si BillBeam detecta un incidente de seguridad, podrá investigar, contener, corregir y tomar medidas razonables según la naturaleza del incidente y la información disponible. {% else %} If BillBeam becomes aware of a security incident, it may investigate, contain, remediate, and take reasonable steps based on the nature of the incident and the information available. {% endif %}

{% if lang == 'es' %} Cuando la ley aplicable lo exija, BillBeam podrá proporcionar avisos apropiados a usuarios o autoridades correspondientes. {% else %} Where required by applicable law, BillBeam may provide appropriate notice to affected users or relevant authorities. {% endif %}

{% if lang == 'es' %}8. Limitaciones{% else %}8. Limitations{% endif %}

{% if lang == 'es' %} Ningún método de transmisión o almacenamiento electrónico puede garantizar seguridad absoluta. Aunque BillBeam busca aplicar medidas razonables, no puede garantizar que el Servicio será completamente inmune a incidentes de seguridad. {% else %} No method of transmission or electronic storage can guarantee absolute security.
